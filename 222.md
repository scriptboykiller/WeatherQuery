问题发生在这里：

jdbcSql.append('?');

bindingValues.add(createBindingValue(
        ++bindingIndex,
        "?" + parameterIndex,
        resolvedParameter,
        effectiveValue(parameterIndex, resolvedParameter, contextHint),
        "Mapped from JPA indexed positional parameter"));

现在它无论参数是 scalar 还是 collection，都只：

追加一个 ?
绑定一个 value

所以 Set<Long> 最后会整体作为 [1] 绑定进去。

最小修改位置

就在你截图中大概第 49～57 行这块，把它替换成“集合展开”。

可以直接改成：

final Object value =
        effectiveValue(parameterIndex, resolvedParameter, contextHint);

if (value instanceof Collection<?> collection && !collection.isEmpty()) {
    int collectionIndex = 0;

    for (final Object item : collection) {
        if (collectionIndex > 0) {
            jdbcSql.append(", ");
        }

        jdbcSql.append('?');

        bindingValues.add(createBindingValue(
                ++bindingIndex,
                "?" + parameterIndex,
                resolvedParameter,
                item,
                "Expanded from JPA indexed collection parameter"));

        collectionIndex++;
    }
} else {
    jdbcSql.append('?');

    bindingValues.add(createBindingValue(
            ++bindingIndex,
            "?" + parameterIndex,
            resolvedParameter,
            value,
            "Mapped from JPA indexed positional parameter"));
}

需要 import：

import java.util.Collection;

这样：

WHERE id IN (?1)

如果 ?1 对应：

Set<Long> = [1, 2, 3]

最终会生成：

WHERE id IN (?, ?, ?)

并产生三条 BindingValue：

1 -> 1L
2 -> 2L
3 -> 3L

还要改 mock 集合大小

你现在 ResolvedParameter 里还是：

collectionSize: 1

所以即便展开，结果仍然只是：

IN (?)

把生成 Collection 的位置从：

mockValueFactory.createCollectionValue(
        elementType,
        parameterName,
        1,
        hint)

改成：

mockValueFactory.createCollectionValue(
        elementType,
        parameterName,
        3,
        hint)

这样报告显示：

mockValue = [1, 2, 3]

SQL 显示：

IN (?, ?, ?)

一个要注意的地方

这个修改适合：

IN (?1)

但如果某个 collection 参数不是放在 IN 里，而是意外出现在普通表达式：

column = ?1

也会展开成：

column = ?, ?, ?

这是错误的。

不过你当前是 MVP，而且这个参数已经明确是：

id in (?1)

所以今晚先这样改没问题。

更严谨的后续版本应当由 SqlParameterContextInferer 判断它是不是 NUMERIC_LIST / STRING_IDENTIFIER_LIST，只有 list context 才展开。
