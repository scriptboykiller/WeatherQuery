看到了，问题就在这里：

for (int index = 0; index < safeSize; index++) {
    values.add(createScalarValue(
            elementType,
            parameterName,
            mockParameterHint));
}

createScalarValue() 对 Long 永远返回同一个 DEFAULT_LONG，所以才得到：

[1, 1, 1]

你现在不用大改 createScalarValue()，只改 createCollectionValue()，这是最小、最安全的做法。

把循环改成：

for (int index = 1; index <= safeSize; index++) {
    values.add(createCollectionElementValue(
            elementType,
            parameterName,
            mockParameterHint,
            index));
}

然后在 MockValueFactory 里增加这个私有方法：

private Object createCollectionElementValue(
        final String elementType,
        final String parameterName,
        final MockParameterHint mockParameterHint,
        final int index) {

    final String normalizedType =
            elementType == null
                    ? ""
                    : elementType.toLowerCase(Locale.ROOT);

    if (normalizedType.contains("long")) {
        return (long) index;
    }

    if (normalizedType.contains("integer")
            || "int".equals(normalizedType)) {
        return index;
    }

    if (normalizedType.contains("short")) {
        return (short) index;
    }

    if (normalizedType.contains("bigdecimal")) {
        return BigDecimal.valueOf(index);
    }

    if (normalizedType.contains("string")
            || mockParameterHint == MockParameterHint.STRING_IDENTIFIER
            || mockParameterHint == MockParameterHint.STRING_IDENTIFIER_LIST) {
        return "TEST_" + index;
    }

    return createScalarValue(
            elementType,
            parameterName,
            mockParameterHint);
}

需要确认已有 imports：

import java.math.BigDecimal;
import java.util.Locale;

修改后：

Set<Long>    -> [1, 2, 3]
List<Integer> -> [1, 2, 3]
List<String> -> [TEST_1, TEST_2, TEST_3]

最终 Binding Plan 应该变成：

bindingIndex  mockValue
1             1
2             2
3             3

而 Execution Report 保持：

WHERE id IN (?, ?, ?)

你现在这个修改只影响集合 mock，不会影响已经跑通的 named scalar、JPA scalar 和普通 JdbcTemplate 参数。
