对，你现在修好的那三个，是 JdbcTemplate 普通 ?：

jdbcTemplate.query(sql, new Object[]{systemName}, ...)

但截图里这些是另一类：

... subscriber_name = ?1
... src_system = ?1
... model_id = ?1

它们属于 Spring Data JPA indexed positional parameter。不能继续用：

arg1
根据名字猜类型
LOW confidence

应该直接按位置读取 Repository 方法签名。

例如截图里：

List<RosettaInterface> findInterfaceBySubscriberNative(String systemName);

SQL 里的：

subscriber_name = ?1

就是：

?1 -> 方法第一个参数 systemName -> String

另外：

List<RosettaInterface> findInterfaceBySourceSystemNative(Integer systemId);

就是：

?1 -> systemId -> Integer

最小修改方案

你现有的 resolvePositionalParameters() 需要同时处理两类：

?1 / ?2  -> Repository 方法参数位置
普通 ?   -> JdbcTemplate new Object[]{...}

1. 增加读取方法签名参数的方法

继续放在 DefaultJavaSourceParameterResolver 中：

private List<PositionalSourceParameter> resolveMethodParameters(
        final Path javaSourceFile,
        final String methodName) {

    try {
        final CompilationUnit compilationUnit =
                StaticJavaParser.parse(javaSourceFile);

        return compilationUnit.findAll(MethodDeclaration.class).stream()
                .filter(method -> method.getNameAsString().equals(methodName))
                .findFirst()
                .map(method -> method.getParameters().stream()
                        .map(parameter -> new PositionalSourceParameter(
                                parameter.getNameAsString(),
                                parameter.getType().asString()))
                        .toList())
                .orElseGet(Collections::emptyList);

    } catch (final Exception exception) {
        LOGGER.warn(
                "Failed to resolve method parameters. file={}, method={}, reason={}",
                javaSourceFile,
                methodName,
                exception.getMessage());

        return Collections.emptyList();
    }
}

继续复用之前的内部 record：

private record PositionalSourceParameter(
        String parameterName,
        String javaType) {
}

2. 判断当前 SQL 是 ?1 还是普通 ?

增加一个简单方法：

private boolean usesIndexedJpaParameters(final NativeSqlRecord record) {
    if (record == null || record.getNormalizedSqlText() == null) {
        return false;
    }

    return record.getNormalizedSqlText().matches("(?s).*\\?\\d+.*");
}

你实际字段如果叫 getSqlText()，替换成对应 getter。

3. 修改 resolvePositionalParameters()

核心结构改成：

private Map<Integer, ResolvedParameter> resolvePositionalParameters(
        final NativeSqlRecord record) {

    if (record == null || record.getParameterCount() <= 0) {
        return Collections.emptyMap();
    }

    final Optional<Path> javaSourceFile =
            projectPathResolver.resolveJavaSourceFile(record);

    final List<PositionalSourceParameter> sourceParameters;

    if (usesIndexedJpaParameters(record)) {
        sourceParameters = javaSourceFile
                .map(path -> resolveMethodParameters(
                        path,
                        record.getMethodName()))
                .orElseGet(Collections::emptyList);
    } else {
        sourceParameters = javaSourceFile
                .map(path -> resolveJdbcTemplateParameters(
                        path,
                        record.getMethodName()))
                .orElseGet(Collections::emptyList);
    }

    final Map<Integer, ResolvedParameter> resolvedParameters =
            new LinkedHashMap<>();

    final int uniqueParameterCount =
            usesIndexedJpaParameters(record)
                    ? findHighestIndexedParameter(record)
                    : record.getParameterCount();

    for (int index = 1; index <= uniqueParameterCount; index++) {

        if (index <= sourceParameters.size()) {
            final PositionalSourceParameter sourceParameter =
                    sourceParameters.get(index - 1);

            resolvedParameters.put(
                    index,
                    createResolvedParameterFromJavaType(
                            sourceParameter.parameterName(),
                            sourceParameter.javaType()));

            continue;
        }

        final String fallbackName = "arg" + index;

        resolvedParameters.put(
                index,
                createResolvedParameter(
                        fallbackName,
                        false));
    }

    return resolvedParameters;
}

4. 正确计算 ?1 / ?2 的唯一参数数量

这个很重要。

例如：

a = ?1 and b = ?1 and c = ?2

parameterCount 可能是 3 次出现，但实际只有两个参数。

增加：

private int findHighestIndexedParameter(final NativeSqlRecord record) {
    final String sql = record.getNormalizedSqlText();

    final Matcher matcher =
            Pattern.compile("\\?(\\d+)").matcher(sql);

    int highestIndex = 0;

    while (matcher.find()) {
        highestIndex = Math.max(
                highestIndex,
                Integer.parseInt(matcher.group(1)));
    }

    return highestIndex;
}

需要：

import java.util.regex.Matcher;
import java.util.regex.Pattern;

这样：

?1、?1、?2

只会生成：

binding parameter 1
binding parameter 2

而不会乱生成三个不同参数。

截图里三条应该得到什么

第一条

findInterfaceBySourceSystemNative(Integer systemId)

结果：

?1
parameterName = systemId
javaType = Integer
mockValue = 1
confidence = HIGH

第二条

findInterfaceBySubscriberNative(String systemName)

结果：

?1
parameterName = systemName
javaType = String
mockValue = TEST
confidence = HIGH

第三条

findInterfaceByDataModelNative(Integer systemId)

结果：

?1
parameterName = systemId
javaType = Integer
mockValue = 1
confidence = HIGH

你现在真正需要的规则

NAMED :name
    -> @Param 或 Java 参数名

INDEXED POSITIONAL ?1/?2
    -> Repository 方法参数位置

ORDINARY POSITIONAL ?
    -> JdbcTemplate new Object[]{...}

全部解析失败
    -> 原来的 LOW confidence fallback

所以你说得对：不能只把新修的三个 JdbcTemplate 做正确，原来的 JPA ?1 也必须全部从真实方法签名拿类型。

这次只增强 resolvePositionalParameters()，不用动 named parameter，也不用改 MockValueFactory。
